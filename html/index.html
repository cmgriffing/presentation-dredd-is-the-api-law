Basics

What is Dredd?

Automated API Documentation Testing

Multi-format Support

  • API Blueprint https://apiblueprint.org/
  • OpenAPI 2 https://swagger.io/

Hooks

Run various functions before or after some or all requests.

  • Token handling
  • Newly created IDs and other State

Hooks (Events Part 1)

  • beforeAll called at the beginning of the whole test run
  • beforeEach called before each HTTP transaction
  • before called before some specific HTTP transaction
  • beforeEachValidation called before each HTTP transaction is validated
  • beforeValidation called before some specific HTTP transaction is validated

Hooks (Events Part 2)

  • after called after some specific HTTP transaction regardless of its result
  • afterEach called after each HTTP transaction
  • afterAll called after the whole test run

Usage

A Basic GET

FORMAT: 1A

# GET /
+ Response 200 (application/json; charset=utf-8)

        {"message": "Hello"}

Dredd will check: Status Code, Content-Type, and provided headers (none used here)

The message key gets checked, but the value does not.

A Basic POST

# POST /api/widgets
+ Request (application/json)

        {"name": "my widget", "size": "large", "weight": "heavy"}

+ Response 200 (application/json; charset=utf-8)

        {"_id": "casd978fasdg98ads7gasdf7g", "name": "my widget", "size": "large", "weight": "heavy"}

The exact request object is sent as the body.

Dynamic Data

# GET /api/widgets/{widgetId}
+ Parameters
    + widgetId: 42 (string, required)

+ Response 200 (application/json; charset=utf-8)

        {"_id": "casd978fasdg98ads7gasdf7g", "name": "my widget", "size": "large", "weight": "heavy"}

Take note of widgetId in url and params.

Using Hooks (handling responses)

const stash = {};

// After creating a widget, add the id from the response to the stash for future use
hooks.after('/widgets > POST', function(transaction) {
  const body = JSON.parse(transaction.real.body);
  stash.widgetId = body._id;
});

We "stash" the widgetId on a higher scoped object

Using Hooks (modifying requests)

// The ID found in the params section of the apib file
const widgetId = '42';

// Use the widgetId from before and parse out the one hardcoded in the docs.
hooks.before('/widgets/{widgetId} > GET', function(transaction) {
  transaction.fullPath = transaction.fullPath.replace(widgetId, stash.widgetId);
  transaction.request.uri = transaction.fullPath;
});

We just modify the transaction instead of returning it. (I would prefer return)

Uploading Images (Hooks)

const hooks = require('hooks');
const fs = require('fs');
const path = require('path');

hooks.beforeEach((transaction, done) => {
  const buffer = fs.readFileSync(path.join(__dirname, '../image.png'));
  transaction.request.body = buffer.toString('base64');
  transaction.request.bodyEncoding = 'base64';
  done();
});

Multiple Requests/Responses (OpenAPI Caveat)

When using OpenAPI 2 format, by default Dredd tests only responses with 2xx status codes.

var hooks = require('hooks');

hooks.before('/resource > GET > 500 > application/json', function (transaction, done) {
  transaction.skip = false;
  done();
});
